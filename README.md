# companion-robot-proteus
A simple Companion Robot simulation using the AT89C51 in Proteus Design Suite. It displays messages on an LM016L LCD and uses a push button to simulate sensor inputs, demonstrating microcontroller-based control and alert systems.


Description
This project is a simplified Companion Robot system designed using the AT89C51 microcontroller. It is implemented and tested in Proteus simulation software.

The system displays messages on an LCD and provides alerts using a buzzer. A push button is used to simulate sensor inputs, demonstrating how real-world sensor signals can be processed by the microcontroller.

⚙️ Features

| LCD message display

| Button-based input simulation

| Buzzer alert system

| Microcontroller-based control

- Working Principle
  
The microcontroller initializes the system

| LCD displays default messages

| Button press simulates input conditions

| Based on input, LCD updates messages

| Buzzer activates for alerts

- Components Used
  

AT89C51 Microcontroller

16×2 LCD (LM016L)

Push Button

Buzzer

Crystal Oscillator

Capacitors

Resistors

Potentiometer (for LCD contrast)


- Software Used

 Proteus Design Suite

Keil uVision (Embedded C)

- Future Scope

| Integration of real sensors (ultrasonic, gas, etc.)
| Real-time monitoring system
| Enhanced automation and interaction
